## 背景

玩家回報：「每天簽到拿到的星幣，沒辦法累計進『每月登入/本月簽到』，所以拿不到獎勵。」

調查確認三層問題：

1. **直接 bug**：`frontend/src/pages/CheckIn.jsx` 簽到後**沒有**把當日日期寫進 localStorage，而 `AppShell.jsx`/`Profile.jsx` 的同名 handler 都有。三頁各自複製貼上同一套 helper、其中一份漏掉 → 從 `/checkin` 頁簽到的日子不會累計進「本月簽到天數/月曆」。
2. **設計脆弱**：月度簽到歷史只存在 localStorage（清快取/換裝置即歸零）；後端 `daily_checkins` 其實有每日紀錄卻沒有端點可讀；mock 連每日日期都沒存，且用 UTC 日界、連續天數只加不重置 → 與後端分歧。
3. **功能缺失**：全專案**沒有**可領取的「月度累計獎勵」。「本月簽到」只是顯示文字，真正發星幣的只有「連續天數里程碑」（連續 7/14/21/30 天）。

---

## 決策

### 1. 新增「月度累計簽到獎勵」（與連續里程碑獨立）
當月**累計**（非連續）簽到天數達里程碑可**手動領取**一次大獎：

| 累計天數 | 星幣 |
|---|---|
| 10 | 2,000 |
| 20 | 5,000 |
| 28（全勤） | 12,000 |

- 領取**僅限當月**（`?month=` 僅供顯示歷史月曆）；達標且未領才可領。
- 發獎照 ADR-002：member 寫 `monthly_reward_claims` 紀錄 + **同交易** `outboxService.save("wallet.credit.request", …)`，wallet 消費後入帳，不在 member 直連 wallet DB。
- 新 credit 子型 **`MONTHLY_REWARD`**（非 `WIN`，避免 rank-service 計入「今日贏幣榜」）。

### 2. 簽到狀態改「後端權威來源」
新增 `GET /api/v1/wallet/checkin/status`（端點在 member-service），回傳 `CheckinStatusResponse{ month, signedDates[], monthCheckinDays, consecutiveDays, checkedInToday, milestones[] }`。前端月曆/本月天數/里程碑領取旗標全部讀此，**移除** localStorage 的 `CHECKIN_DATES_KEY` 與三頁重複的 date helper（保留純 UI 的 `CHECKIN_AUTO_OPEN_KEY`）。前端統一改用 `hooks/useDailyCheckIn.js` + `utils/checkInDates.js`，從根本消除「某頁 handler 漏存」類 bug。

### 3. 冪等（雙重）
- 應用層：claim 前 `existsByPlayerIdAndRewardMonthAndMilestoneDays`。
- DB：`monthly_reward_claims` 的 `UNIQUE(player_id, reward_month, milestone_days)` + `wallet_transactions.idempotency_key` UNIQUE，`idempotencyKey = monthly-reward-{playerId}-{yyyy-MM}-{milestoneDays}`。

## 契約（單一真相）

| 項目 | 值 |
|---|---|
| 狀態端點 | `GET /api/v1/wallet/checkin/status`（可選 `?month=YYYY-MM`，預設 `YearMonth.now(Asia/Taipei)`） |
| 領獎端點 | `POST /api/v1/wallet/checkin/monthly-reward`，body `{ "milestoneDays": 10 }` |
| 子型 | `MONTHLY_REWARD` |
| 里程碑 | `10→2000`、`20→5000`、`28→12000` |
| 錯誤碼 | 無效里程碑 400 / 未達標 422 / 已領取 409 |

## 後果與注意

- **新增 credit 子型須四同步**（見 AGENTS.md 雷區）：`CreditRequest.@Pattern` + MySQL/Postgres 的 `chk_wt_sub_type`（init.sql 與 migration）。
- **member-service 的 `/api/v1/wallet/**` 子路徑必須加進 gateway `member-checkin` 路由 Path**（排在 wallet catch-all 之前），否則被轉到 wallet-service → 404。
- **member 是 `ddl-auto: validate`**：`monthly_reward_claims` 欄位須與 entity 完全吻合；`reward_month` 刻意避開 MySQL 關鍵字 `YEAR_MONTH`。
- 跑真實後端前須套用 MySQL `V9` / Postgres `V12` migration。
