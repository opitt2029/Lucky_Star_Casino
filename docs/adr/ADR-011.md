# ADR-011 — 第三方登入採 OIDC subject 綁定與一次性票據交換

| 項目 | 內容 |
|------|------|
| **狀態** | ✅ 已接受（Accepted） |
| **日期** | 2026-07-23 |
| **影響範圍** | Member Service、玩家前端、MySQL、Redis |

---

## 背景

登入頁已有 LINE、Google、Apple 按鈕，會員中心也有社群綁定畫面，但原本只回傳
`demo-*` 資料，沒有向供應商授權、驗證身分或持久化綁定。

OAuth callback 若直接把 Lucky Star access/refresh token 放在 URL，token 會進入瀏覽器歷史、
代理日誌與 Referer；若只用 email 合併帳戶，也可能造成帳戶接管。

## 決策

1. 使用 Spring Security OAuth2 Client 執行 authorization-code / OIDC 流程。
2. 第三方身分以供應商簽發的 `(provider, sub)` 為唯一鍵，不以 email 作主鍵。
3. 第三方登入只接受已綁定帳戶；既有 email 相同也不自動合併，玩家必須先用帳密登入後綁定。
4. OAuth 成功後，Member Service 將 Lucky Star JWT 暫存 Redis，前端只收到 2 分鐘、
   使用後立即刪除的一次性 ticket，再以 POST 交換 token。
5. 綁定起始 ticket TTL 為 5 分鐘，且由已驗證玩家 API 建立；callback 不接受前端自行提交
   `externalAccountId`。
6. OAuth endpoint 全放在 `/api/v1/auth/**`，沿用 Gateway 既有路由與白名單。

## 資料模型

`member_social_accounts`：

- `UNIQUE(provider, provider_subject)`：同一第三方帳戶不能綁兩個玩家。
- `UNIQUE(member_id, provider)`：每位玩家每個 provider 只能綁一個帳戶。
- `member_id` 外鍵 `ON DELETE CASCADE`。

## 後果

- 未設定 Client ID/Secret 時 provider 預設停用，服務仍可啟動。
- Apple `client_secret` 必須是以 Apple 私鑰簽出的 JWT，需依 Apple 有效期輪替。
- Apple Web Login callback 必須使用公開 HTTPS 網域，不能使用 `localhost` 或 IP；本機整合測試需透過已登記網域或 HTTPS tunnel。
- OAuth 握手會短暫使用 HttpSession 保存 `state`；callback 完成或失敗後立即銷毀，
  一般 API 仍以 JWT 驗證。
