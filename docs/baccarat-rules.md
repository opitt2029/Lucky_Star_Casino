# 百家樂遊戲規則（T-034）

> 對應實作：`backend/game-service/.../baccarat/BaccaratGameService.java`
> 本文件為交付物之一，描述本系統採用的標準百家樂（Punto Banco）規則與派彩設計，供前端、
> 測試與稽核對齊。

---

## 1. 點數計算

- A = 1 點；2~9 = 面值；10 / J / Q / K = 0 點。
- 一手牌的點數 = 各牌點數總和「取個位數」（即 `總和 mod 10`）。
  - 例：7 + 8 = 15 → 5 點；K + 9 = 0 + 9 = 9 點。

## 2. 發牌與牌局來源

- 押注區有三個：**閒（PLAYER）／莊（BANKER）／和（TIE）**。
- 發牌順序：**閒1 → 莊1 → 閒2 → 莊2**，必要時依補牌規則各補第三張（最多 3 張）。
- 牌由 Provably Fair RNG 串流（`SHA-256(serverSeed:clientSeed:nonce:block)`）確定性產生：
  每張牌先抽 `nextInt(13)` 決定牌面、再抽 `nextInt(4)` 決定花色。相同 `(serverSeed, clientSeed, nonce)`
  必得相同牌局，玩家可事後重算驗證（見 T-036）。
- **設計取捨——無限靴模型**：每張牌獨立、均勻、可重複抽取（不模擬有限靴的抽出不放回）。
  此舉讓各牌面等機率、實作與驗證單純；對莊/閒/和的理論機率影響極小，且不影響公平性可驗證性。

## 3. 補牌規則（標準 Punto Banco）

### 3.1 天牌（Natural）
- 若**閒或莊**的前兩張點數為 **8 或 9**，雙方皆**停牌**，直接比點。

### 3.2 閒家補牌
- 無天牌時：閒家點數 **0~5 補**第三張，**6~7 停**。

### 3.3 莊家補牌
- **若閒家未補牌**（停在 6~7）：莊家比照閒家規則，**0~5 補、6~7 停**。
- **若閒家補了第三張**（其點數記為 `p3`）：依下表決定莊家是否補牌：

| 莊家點數 | 補牌條件 |
|---|---|
| 0、1、2 | 一律補 |
| 3 | 補，除非 `p3 = 8` |
| 4 | `p3 ∈ {2,3,4,5,6,7}` 才補 |
| 5 | `p3 ∈ {4,5,6,7}` 才補 |
| 6 | `p3 ∈ {6,7}` 才補 |
| 7 | 停牌 |

## 4. 勝負判定

- 補牌完成後比較雙方點數：點數大者勝；相同則為 **和（TIE）**。

## 5. 派彩（含本金返還）

設某押注區下注額為 `bet`：

| 押注區 | 結果 | 派彩（含本金） | 淨利 |
|---|---|---|---|
| 閒 PLAYER | 閒贏 | `bet × 2` | +bet（1:1） |
| 莊 BANKER | 莊贏 | `bet × 2 − floor(bet × 5%)` | +bet × 0.95（扣 5% 傭金） |
| 和 TIE | 和局 | `bet × 9` | +bet × 8（8:1） |
| 閒／莊 | 和局 | `bet`（退回本金，push） | 0 |
| 任一 | 押錯 | `0` | −bet |

- **莊贏 5% 傭金**：以「贏額」的 5% 計傭，整數向下取整（`floor`）。例：押莊 100、莊贏 →
  派 `200 − floor(100 × 0.05)=200 − 5 = 195`，淨利 +95。
- **和局退本金**：和局時押莊／押閒不算輸，退回本金（push）。押和則依 8:1 派彩。
- 金額皆為整數「星幣」；傭金取整採向下取整，對玩家無不利之外的額外損耗。
