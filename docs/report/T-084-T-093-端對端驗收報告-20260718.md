### 3.3 驗收過程發現並修復的問題（vite dev proxy 剝除 Origin）

實測初輪 UI 登入即 403：dev server 跑在 5318，瀏覽器對**同源** POST 仍會帶
`Origin: http://localhost:5318`，vite proxy 原樣轉發 → gateway CORS 白名單只認 5173/5174 → 403 拒絕。
第二輪修 HTTP 後 WS 升級仍卡：http-proxy 的 `proxyReq` 事件只涵蓋一般 HTTP（SockJS `/ws/info`），
**WS upgrade 走 `proxyReqWs`**，Origin 沒剝到照樣被擋。

修復（`frontend/vite.config.js`）：`/api` 與 `/ws` proxy 的 `configure` 掛 `proxyReq`＋`proxyReqWs`
一律 `removeHeader('origin')`——經 dev proxy 的請求對瀏覽器本就是同源，轉發瀏覽器 Origin 給 gateway
反而誤導其 CORS 判定；剝除後 gateway 視為一般非 CORS 請求，任何 dev/e2e port 都通用
（比照 frontend-admin 以 vite proxy 免碰 CORS 白名單的既有作法）。
此修復對既有流程無影響：預設 dev（5173）直連 8080 不走 proxy；mock e2e 不打真後端。
